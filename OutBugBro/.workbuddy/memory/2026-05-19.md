## 手写.tscn全面失败 → 改为程序化构建
- **根因**: Godot 4.5 手写 .tscn 格式极不可靠（#注释被当颜色代码、uid缺失、ext_resource id格式等），多次尝试不同格式均无法解析
- **踩坑**: .tscn 中不能用 `# 注释`（被解析为颜色代码）；sub_resource id 用描述性字符串可能失败；缺少 uid 可能失败
- **方案**: 删掉7个坏.tscn，全部改为程序化构建
  - Enemy基类: `_visual_params()` 虚方法返回外观参数 + `_build_visual()` 创建 LevelLabel/HpBar/Collision/Body
  - 子类覆写 `_visual_params()` 即可自定义外观（FastEnemy绿/TankEnemy紫/RangedEnemy橙/Boss红）
  - EnemyBullet: `EnemyBullet.create(pos, dir, dmg)` 静态工厂方法
  - Boss: `Boss.new()` 直接创建，不再需 .tscn
  - EnemySpawner: `Enemy.new()` / `FastEnemy.new()` 等直接 new
  - TargetHpBar / BossHpBar: 代码构建 UI，不再依赖 .tscn
  - main.tscn: TargetHpBar 和 BossHpBar 改为脚本节点(MarginContainer + script)
- **保留的.tscn**: 只有 enemy_a.tscn（Godot编辑器创建的，有正确uid）和其他已有.tscn
- **load_steps 踩坑**: enemy_spawner.gd 中 `preload` 改为 `load` 无效（因为.tscn本身就解析不了），最终删掉.tscn
