## 掉落表修复（第二轮）
- **根因**: 手写 `default_drop_table.tres` 使用 `[sub_resource type="DropEntry"]`，Godot 4 无法在运行时解析自定义 Resource 类型，报 `Cannot get class 'DropEntry'`
- **每次都走程序化构建**，.tres 里改的权重（如 castorice=50）完全不生效
- **修复**: 删除无法加载的 .tres，让程序化构建 `_build_default_drop_table()` 成为唯一数据源
- **权重同步**: 程序化构建权重与 .tres 中的一致（castorice=50），后续改权重直接改 `_build_default_drop_table()` 中的字典
- **添加掉落调试日志**: 每个怪物死亡时输出 drop_chance/elite/掉落结果，方便排查
