# 项目总览 — 游戏设计与核心系统

> 本文档面向所有项目成员，概述游戏玩法、核心系统和设计思路。

---

## 游戏概述

玩家操控一个固定炮台，面对无尽波次的怪物。通过拾取掉落物品、升级属性、购买技能来提升战斗力，尽可能撑到更高波次获得更高评分。

**核心循环**: 击杀怪物 → 获得金币 + 物品 → 升级/购买技能 → 挑战更高波次

---

## 核心系统

### 射击系统

- **AUTO 模式**: 自动瞄准最近敌人，lerp 旋转炮管
- **MOUSE 模式** (Shift 切换): 鼠标瞄准，瞬间朝向
  - 左键/右键均可触发蓄力
  - 0.3~1.5s：普通蓄力（2-3x 伤害，穿透 3 目标）
  - 1.5~5.0s：重蓄力（5-7x 伤害，穿透 5 目标，W7+ 解锁）
  - 蓄力弹 W5+ 有爆炸溅射
  - 重蓄力触发屏幕轻微抖动
  - 鼠标左键点击 = 普通射击，按住 = 蓄力

### 波次系统

- **FIGHTING → BREAK → FIGHTING** 状态机
- 每波 30 秒战斗，5 秒休息期，清波金币奖励
- 精英怪：10% 概率，HP×3，体型 1.5 倍，金色外观，必掉稀有+物品
- 生成间隔随波次加快（最快 1.6 秒），最多 10 只同屏

**难度阶段**:

| 阶段 | 波次 | 颜色 |
|------|------|------|
| 轻松 | 1~3 | 绿色 |
| 普通 | 4~10 | 蓝色 |
| 困难 | 11+ | 红色 |

### 掉落系统

- 加权随机掉落表（程序化构建，8 种物品）
- 品质视觉反馈：传说→金色光柱，史诗→紫色脉冲，稀有→青色闪烁
- 连击加成最高 +50% 掉率

**品质等级**:

| 等级 | 品质 |
|------|------|
| 0 | 普通（白） |
| 1 | 精良（绿） |
| 2 | 稀有（青） |
| 3 | 史诗（紫） |
| 4 | 传说（金） |

### 技能系统（共 7 个）

| 技能 | 解锁 | 费用 | CD | 效果 |
|------|------|------|-----|------|
| 临时护盾 | W3 | 1000g | 20s | 5s 免疫伤害 |
| 主动防御 | W7 | 1500g | 80s | 8s 内 75% 反伤 + 80% 减伤 |
| 应急修理 | W10 | 4500g | 120s | 回血 90% + 10s 极大属性增幅 |
| 制导导弹 | W15 | 18000g | 200s | 全屏清怪 + 20s 即死光环 |
| 头油 | 免费 | - | 30s | 击退+减速附近敌人 |
| 双发 | W10 | 10000g | 60s | 60s 内蓄力弹双发 |
| 磁力拉拽 | W8 | 5000g | 30s | 15s 内蓄力时间减半+敌人聚拢 |

### 位移技能（Q 键）

独立于技能系统的免费位移，自动挂载到玩家：

- 速度: 800px/s，持续 0.15s
- CD: 10s，击杀减 1s CD
- 方向: 鼠标方向

### 存档系统

- 自动存档（每波休息期）
- 存档内容：波次/金币/背包/技能解锁/属性升级/最高分
- 死亡只更新最高分，其余清零
- 主菜单支持继续游戏/新游戏

### 评分系统

| 评分 | 条件（波次） |
|------|------------|
| C | < 6 波 |
| B | 6 波 |
| A | 7~8 波 |
| S | 9~11 波 |
| SS | 12 波 |
| SSS | 13 波 |
| SSS+ | 14 波 |
| SSS+ ⭐×N | ≥15 波（每多 1 波 +1 星） |

### 挑战模式

独立 Boss 战模式，从主菜单进入：

- Boss：222 级大虫
- 限时：300 秒
- 血包：99 个
- 技能：仅位移（Q 键）
- 评分公式：`造成伤害 / max(受到伤害, 1)`，击杀 Boss×2

## 目录结构

```
core/
  autoload/     → 8 个全局单例
  base/         → 基类（Component/ItemData/EffectData/DropTable 等）
  components/   → 8 个组件（Health/Move/Shooting/Dash 等）
data/
  effects/      → 11 个效果 .tres
  items/        → 12 个物品 .tres
  skills/       → 7 个技能 .tres
  drop_tables/  → 掉落表
scenes/
  world/        → 核心游戏场景
  enemies/      → 5 种敌人类型
  ui/           → 所有 UI 场景
  challenge/    → 挑战模式
assets/         → SVG 图标 + 精灵
addons/
  godot_mcp/    → Godot MCP 编辑器插件
```

## 性能优化

| 优化项 | 方法 |
|--------|------|
| 子弹拖尾 | 静态对象池（80 个 ColorRect 复用） |
| 伤害数字 | 静态对象池（20 个 Label 复用） |
| 拖尾组件 | 全局静态池 60 个，ACTIVE_MAX=15 |
| 存活计数 | `_enemies_alive` 计数器替代每帧 `get_nodes_in_group()` |

## 设计原则

1. **EventBus** 解耦通信，禁止硬引用跨模块
2. **组件组合** 模式，实体用组件子节点，不用继承
3. **数据驱动** — .tres Resource 管理内容，新增物品/效果零代码
4. **DataRegistry** 统一管理资源路径
5. 命名 `snake_case`，中文注释
